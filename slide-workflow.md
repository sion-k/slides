# 슬라이드 작업 규칙

## 공통
- 각 문제 파일은 `\setproblem{번호}`로 시작한다.
- 첫 슬라이드는 제목을 포함한 문제 지문 슬라이드다.
- 지문 슬라이드는 `문제` block 하나만 사용한다.
- `입력`, `출력`, `제한` block은 만들지 않는다. 대신 제한은 변수가 처음 등장할 때 `N($1 \le N \le ...$)`처럼 넣는다.
- 예제는 `예제 입력/출력 n` 프레임으로 만든다.
- 예제는 좌측 입력, 우측 출력 2단 컬럼으로 배치한다.

## 소스코드
- 소스코드 슬라이드는 문서 맨 아래에 추가한다.
- 형식은 `\begin{frame}[fragile, allowframebreaks]{이름 소스 코드}`를 사용한다.
- `src/문제번호` 아래 실제 존재하는 파일만 추가한다.
- 파일명이 사람마다 다를 수 있으니 실제 경로를 확인한다.
- 기본 순서는 `김시온`, `원찬혁`, `손현준`, `김지훈`, `김준형`, `서민종`, `윤정`, `임건애`, `정의찬`이다. 없는 사람은 생략한다.
- 한 사람이 여러 풀이를 갖고 있으면 `(BottomUp)`처럼 구분해서 각각 넣는다.
